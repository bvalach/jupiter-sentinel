# Sentinel — Phase 4: Merge & Report (Claude)

You are generating the daily operator briefing by merging:
- Deterministic scanner findings from `scan-summary.json`
- External threat findings from `opus-findings.json`
- Internal posture findings from `codex-findings.json`

Also review yesterday's summary or report if present under `{{AUDIT_DIR}}/{{YESTERDAY}}/`.

## Your tasks

### 1. Merge and deduplicate
- Treat `scan-summary.json.preliminary_findings` as the deterministic baseline
- Fold in analyst findings without duplicating the same issue
- Preserve the most actionable phrasing

### 2. Prioritize
Use these meanings:
- `critical`: requires operator action today
- `high`: serious risk to address this week
- `medium`: should be addressed, not emergency
- `low` / `info`: awareness or hygiene

### 3. Generate report
Write `{{AUDIT_DIR}}/{{TODAY}}/report.md` in this exact format:

```markdown
# Sentinel Daily Brief — {{TODAY_ISO}}
**Host:** {{HOSTNAME_LABEL}}
**Status:** [emoji] CRITICAL / ELEVATED / NORMAL / ALL CLEAR

## Traffic light
- Red (action today): N findings
- Amber (action this week): N findings
- Green (monitor): N findings

## Top 3 actions
1. [CRITICAL/HIGH] Title — what to do (1 line)
2. ...
3. ...

## Changes since yesterday
- [+] New finding: ...
- [-] Resolved: ...
- [~] Changed: ...
(or "No changes" if clean)

## Coverage
- Scanner: ok / degraded
- Threat intel: ok / degraded / skipped
- Forensic audit: ok / degraded / skipped

## External threats
- Threat title — relevance to this system (1 line)
(or "No relevant external threats in last 72h")

## Full findings

### Critical
| ID | Category | Title | Recommendation |
| --- | --- | --- | --- |

### High
| ID | Category | Title | Recommendation |
| --- | --- | --- | --- |

### Medium & Low
(collapsed summary)

---
*Generated by Sentinel at TIMESTAMP. Deterministic scanner + Claude threat intel + Codex forensic audit.*
```

### 4. Write machine-readable summary
After writing the report, write `{{AUDIT_DIR}}/{{TODAY}}/summary.json`:

```json
{
  "date": "YYYY-MM-DD",
  "hostname": "{{HOSTNAME_LABEL}}",
  "status": "critical|elevated|normal|all_clear",
  "counts": {
    "critical": 0,
    "high": 0,
    "medium": 0,
    "low": 0,
    "info": 0
  },
  "top_actions": ["action 1", "action 2", "action 3"],
  "delta_from_yesterday": {
    "new": 0,
    "resolved": 0,
    "changed": 0
  },
  "coverage": {
    "scanner": "ok|degraded",
    "threat_intel": "ok|degraded|skipped",
    "forensic_audit": "ok|degraded|skipped"
  },
  "all_findings": [
    "... merged and deduplicated findings ..."
  ]
}
```

## Rules
- Maximum 1 page for the human-readable report. Dense, not verbose.
- If everything is clean, say so in 5 lines.
- Do not re-analyze the live system. Only synthesize `scan-summary.json`, analyst findings, and yesterday's output.
